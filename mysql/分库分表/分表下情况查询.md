##  分表之后做“范围查询”

分表后做范围查询（18-29 岁）完整方案

你现在是**按用户 ID / 哈希分表**（user_0 ~ user_3），这种分表方式**天然不支持跨表范围查询**（年龄、时间、区间都算）

现在的分表规则：

`user_0`、`user_1`、`user_2`、`user_3` = **按用户 ID 取模 / 哈希分片**

特点：

- 同一个用户 → 固定在一张表
- **年龄相同的用户，会随机散落在所有分表里**

所以：

**直接查单张表查不全，必须查所有表**

### 方案 1：最简单 → 全表扫描 + 合并（小数据量首选）

适用场景

数据量不大（单表 < 100w）、并发不高、快速上线

实现逻辑（伪代码）

1. 遍历所有分表：user_0、user_1、user_2、user_3
2. 每张表都执行 `age between 18 and 29`
3. 代码层合并结果、排序、分页

SQL 示例（单表）

```sql
SELECT * FROM user_0 WHERE age BETWEEN 18 AND 29;
SELECT * FROM user_1 WHERE age BETWEEN 18 AND 29;
SELECT * FROM user_2 WHERE age BETWEEN 18 AND 29;
SELECT * FROM user_3 WHERE age BETWEEN 18 AND 29;
```

代码层合并（Java/Python/Go 都一样）

```php
结果集 = []
for 表名 in [user_0, user_1, user_2, user_3]:
    数据 = 查询(表名, age between 18 and 29)
    结果集.extend(数据)

结果集.sort(创建时间/ID)
返回 分页(结果集, pageNum, pageSize)
```

优点

- 零改造
- 最快上线

缺点

- 数据量大后性能差
- 分页很难精确（必须查全量再分页）

#### 代码示例

简单版（遍历所有分表 + 合并结果）

数据量不大、快速上线

```php
<?php
// 数据库配置
$pdo = new PDO("mysql:host=127.0.0.1;dbname=test;charset=utf8mb4", "root", "");

// 要查询的分表
$tables = ['user_0', 'user_1', 'user_2', 'user_3'];

// 范围条件
$minAge = 18;
$maxAge = 29;

$result = [];

// 遍历所有分表查询
foreach ($tables as $table) {
    $stmt = $pdo->prepare("SELECT * FROM {$table} WHERE age BETWEEN ? AND ?");
    $stmt->execute([$minAge, $maxAge]);
    $list = $stmt->fetchAll(PDO::FETCH_ASSOC);
    
    // 合并结果
    $result = array_merge($result, $list);
}

// 排序（可选）
array_sort($result, 'create_time', SORT_DESC);

// 输出最终结果
var_dump($result);

// 二维数组排序函数
function array_sort(&$array, $key, $sort = SORT_ASC) {
    $column = array_column($array, $key);
    array_multisort($column, $sort, $array);
}
?>
```

优点

- 0 改造表结构
- 5 分钟上线
- 代码简单易懂

缺点

数据量大后变慢（百万级以下没问题

### 方案 2：最优通用方案 → 建立**倒排索引表**（生产推荐）

核心思路

专门建一张**索引表**，把**年龄 → 用户 ID**映射存起来

```sql
user_age_index (
    age int,
    user_id bigint,
    primary key (age, user_id)
)
```

查询流程（18-29 岁）

1. **查索引表**：获取 18~29 岁所有 user_id
2. **路由分表**：根据 user_id 计算落在 user_0 ~ user_3 哪张表
3. **精确查询**：只查对应表，性能极高

优势

- 范围查询 = 索引查询 + 精准单表查询
- 性能接近不分表
- 支持分页、排序、count 精确计算

适合

90% 互联网公司分表场景的标准方案

####  代码示例

生产推荐（倒排索引表）

第 1 步：建索引表（只需执行一次）

```sql
CREATE TABLE user_age_index (
    age INT NOT NULL,
    user_id BIGINT NOT NULL,
    PRIMARY KEY (age, user_id),
    KEY idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

第 2 步：PHP 查询代码（性能极高）

```php
<?php
$pdo = new PDO("mysql:host=127.0.0.1;dbname=test;charset=utf8mb4", "root", "");

$minAge = 18;
$maxAge = 29;

// 1. 从索引表拿到符合年龄的 user_id
$stmt = $pdo->prepare("SELECT user_id FROM user_age_index WHERE age BETWEEN ? AND ?");
$stmt->execute([$minAge, $maxAge]);
$userIds = $stmt->fetchAll(PDO::FETCH_COLUMN, 0);

if (empty($userIds)) {
    echo "无数据";
    exit;
}

// 2. 根据 user_id 路由到对应的分表
$tableMap = [];
foreach ($userIds as $uid) {
    $tableIndex = $uid % 4;        // 你是 4 张分表
    $table = "user_$tableIndex";
    $tableMap[$table][] = $uid;
}

// 3. 批量查询每张表
$result = [];
foreach ($tableMap as $table => $uids) {
    $ids = implode(',', $uids);
    $stmt = $pdo->query("SELECT * FROM {$table} WHERE user_id IN ($ids)");
    $list = $stmt->fetchAll(PDO::FETCH_ASSOC);
    $result = array_merge($result, $list);
}

var_dump($result);
?>
```

优点

- **千万级数据也飞快**
- 支持分页、排序、count
- 生产标准方案

你必须加的索引（非常重要）

```sql
ALTER TABLE user_0 ADD INDEX idx_age(age);
ALTER TABLE user_1 ADD INDEX idx_age(age);
ALTER TABLE user_2 ADD INDEX idx_age(age);
ALTER TABLE user_3 ADD INDEX idx_age(age);
```

总结

- **小数据量** → 用方案 1（复制就能跑）
- **上线 / 正式项目** → 用方案 2（性能最好）
- 两套代码都是**PHP 原生、可直接部署**





### 方案 3：高性能方案 → 使用**ElasticSearch**

适用场景

- 大量范围查询（年龄、性别、城市、时间）
- 复杂筛选组合
- 大数据量（千万～亿级）

流程

1. 把 user 表数据同步到 ES
2. ES 做 `age:[18 TO 29]` 范围查询
3. 拿到 user_id 后回查分表获取详情

优点

- 极快的范围 / 多条件查询
- 天然支持分页、聚合、排序

缺点

- 增加中间件
- 有数据同步延迟

快速选型建议（直接照抄）

|   数据量    |     方案     | 难度 | 性能 |
| :---------: | :----------: | :--: | :--: |
|  < 100 万   | 全表扫描合并 | 极低 | 一般 |
| 100~1000 万 |  倒排索引表  |  中  | 优秀 |
|  > 1000 万  |  ES + 分表   |  高  | 顶级 |

关键坑点提醒（非常重要）

1. **不要在分表上直接跨表分页**（limit/offset）

   跨表分页必须**先查所有表 → 合并 → 内存排序 → 分页**

   

2. **必须给 age 字段加索引**

   每张 user 表都要建：

   ```sql
   CREATE INDEX idx_age ON user_0(age);
   ```

   

3. **范围查询不能用分片键优化**

   按 ID 哈希分表 = 任何非 ID 范围查询都必须查全部分表（或走索引表）
