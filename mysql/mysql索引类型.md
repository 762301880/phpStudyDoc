## 一、按「约束 / 功能」分类（最常用）

### 1. 主键索引 (PRIMARY KEY)

- **定义**：表的主键自动创建的索引，**唯一且非空**，一个表只能有一个。
- **特点**：最快、最核心的索引，数据行物理存储按主键排序（InnoDB）。
- **示例**：`id INT PRIMARY KEY`

### 2. 唯一索引 (UNIQUE)

- **定义**：索引列的值**必须唯一**，允许 NULL（但只能有一个 NULL）。
- **特点**：保证数据唯一性，同时加速查询。
- **示例**：`UNIQUE INDEX idx_phone (phone)`

### 3. 普通索引 (INDEX / KEY)

- **定义**：最基础的索引，**无任何约束**，仅用于加速查询。
- **特点**：最常用，支持重复值、NULL。
- **示例**：`INDEX idx_name (name)`

### 4. 全文索引 (FULLTEXT)

- **定义**：专门用于**文本内容搜索**（如文章、评论关键词检索）。
- **特点**：比 `LIKE %关键词%` 快得多，仅支持 CHAR/VARCHAR/TEXT 类型。
- **示例**：`FULLTEXT INDEX idx_content (content)`

------

## 二、按「字段数量」分类

### 1. 单值索引

- 只包含**一个字段**的索引（上面的主键、唯一、普通索引大多是单值）。

### 2. 联合索引（复合索引）

- **定义**：包含**多个字段**的索引，遵循**最左前缀原则**。
- **核心规则**：查询时必须从索引最左侧字段开始匹配，否则索引失效。
- **示例**：`INDEX idx_user_age_gender (age, gender)`
- **适用**：多条件组合查询（如 `where age=20 and gender=1`）。

------

## 三、按「物理存储」分类（InnoDB 核心）

### 1. 聚簇索引 (Clustered Index)

- **定义**：**主键索引就是聚簇索引**，数据行直接存在索引叶子节点。
- **特点**：查询速度极快，一个表只有一个。
- **注意**：没有主键时，MySQL 会自动用唯一键 / 隐藏字段当聚簇索引。

### 2. 非聚簇索引 (Secondary Index)

- **定义**：除主键外的所有索引（普通、唯一、联合等）。
- **特点**：叶子节点存储**主键值**，查询时需要回表查完整数据。

------

## 四、快速总结（记忆版）

1. **必用**：主键索引（自带，无需手动建）
2. **保唯一**：唯一索引（手机号、邮箱）
3. **加速查询**：普通索引（高频查询字段）
4. **文本搜索**：全文索引（文章、内容）
5. **多条件查询**：联合索引（最左前缀原则）
6. **存储结构**：聚簇索引（主键）+ 非聚簇索引（其他）

### 总结

- 按功能分：**主键、唯一、普通、全文**
- 按字段分：**单值、联合**
- 按存储分：**聚簇、非聚簇**
- 日常开发 90% 场景只用：**主键索引 + 普通索引 + 联合索引**
