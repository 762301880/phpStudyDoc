## [MoonTVPlus ](https://github.com/mtvpls/MoonTVPlus)

### 一、整体架构

- MoonTVPlus（Next.js 静态 / SSG 应用）→ 部署到 **Netlify**

- 存储二选一：

  - **Redis（推荐）**：用 Upstash（Serverless Redis，免费额度够用）
  - **MySQL**：用 TiDB Cloud/PlanetScale（Serverless MySQL，支持 SSL）

  

- 数据通过环境变量连接，无需后端服务器。

------

### 二、准备工作

1. **GitHub 账号**：Fork 官方仓库（https://github.com/mtvpls/moontvplus）

2. **Netlify 账号**：https://netlify.com（免费）

3. 存储账号（二选一）

   - Redis：https://upstash.com（免费）
   - MySQL：https://tidbcloud.com（免费）

   

------

### 三、Netlify 基础部署（默认 LocalStorage）

1. Fork 仓库到自己 GitHub

2. 登录 Netlify → **Add new project** → **Import an existing project**

3. 授权 GitHub，选择 Fork 的仓库

4. 环境变量（基础）：

   ```
   PASSWORD=你的管理员密码
   USERNAME=admin
   ```

   

5. 保持默认构建配置（Netlify 自动识别 Next.js）

6. 点击 **Deploy**，等待 2–3 分钟完成。

------

### 四、Redis（Upstash）配置（推荐）

1. 登录 Upstash → **Create Database** → 选 Redis

2. 复制：

   - **HTTPS ENDPOINT**（如：https://xxx.upstash.io）
   - **TOKEN**（长字符串）

   

3. Netlify 项目 → Site settings → Environment variables

   

   添加：

   ```shell
   NEXT_PUBLIC_STORAGE_TYPE=upstash
   UPSTASH_URL=https://xxx.upstash.io
   UPSTASH_TOKEN=你的token
   ```

   

4. 触发重新部署（Push 到 main 或 Netlify 中手动 Redeploy）。

------

### 五、MySQL（TiDB Cloud）配置

1. 登录 TiDB Cloud → **Create Cluster** → 选 Serverless（免费）

2. 获取连接信息：

   - Host: [xxx.tidbcloud.com](https://xxx.tidbcloud.com)
   - Port: 4000
   - User: root
   - Password: 你的密码
   - Database: test

   

3. Netlify 环境变量：

   ```shell
   NEXT_PUBLIC_STORAGE_TYPE=mysql
   DATABASE_URL=mysql://root:密码@xxx.tidbcloud.com:4000/test?sslaccept=strict
   ```

   

4. 重新部署（注意：MySQL 需开启 SSL，否则 Netlify 无法连接）。

------

### 六、关键环境变量汇总

|           变量           | Redis（Upstash） | MySQL（TiDB） |
| :----------------------: | :--------------: | :-----------: |
| NEXT_PUBLIC_STORAGE_TYPE |     upstash      |     mysql     |
|       UPSTASH_URL        |        ✅         |       ❌       |
|      UPSTASH_TOKEN       |        ✅         |       ❌       |
|       DATABASE_URL       |        ❌         |       ✅       |
|         USERNAME         |        ✅         |       ✅       |
|         PASSWORD         |        ✅         |       ✅       |

------

### 七、验证与访问

1. 部署完成后，Netlify 分配域名（如：xxx.netlify.app）
2. 访问 → 进入 MoonTVPlus，数据会自动存入 Redis/MySQL
3. 绑定自定义域名（可选）：Netlify → **Domain settings**。

------

### 八、常见问题

- **部署失败**：检查环境变量是否正确，无空格 / 引号
- **Redis 连接失败**：确认 Upstash 的 ENDPOINT 和 TOKEN 正确
- **MySQL 连接失败**：必须加 `?sslaccept=strict`，否则 Netlify 拒绝非 SSL 连接。

## [synctv](https://github.com/synctv-org/synctv)

**docker-compose搭建**

```php
version: '3.3'
services:
  synctv:
    image: 'synctvorg/synctv:latest'
    container_name: synctv
    restart: unless-stopped
    ports:
      - '8081:8080/tcp'
      - '8081:8080/udp'
    volumes:
      - /opt/synctv:/root/.synctv
    environment:
      - PUID=0
      - PGID=0
      - UMASK=022
      - TZ=Asia/Shanghai
```

### Linux 一步到位（Cloudflare Tunnel

① 下载 cloudflared

直接用 curl：

```shell
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o cloudflared
```

② 加执行权限

```shell
chmod +x cloudflared
```

③ （可选）放到全局

这样你以后随便在哪都能用：

```shell
sudo mv cloudflared /usr/local/bin/
```

④ 启动你的项目

比如：

```shell
npm run dev
```

假设你是：

```shell
http://localhost:5173
```

⑤ 开隧道（关键一步）

```shell
cloudflared tunnel --url http://localhost:5173

# 补充 自动常驻后台 -使用nohub命令

nohup cloudflared tunnel --url http://localhost:8081 > cf.log 2>&1 &

# 杀死进程
root@iZwz9f5r5auq14qsnsd44fZ:~# ps aux | grep cloudfla
root     4030619  0.2  2.2 1260800 37504 pts/1   Sl   18:23   0:00 cloudflared tunnel --url http://localhost:5173
root     4030724  0.0  0.1   6544  2304 pts/1    S+   18:24   0:00 grep --color=auto cloudfla
root@iZwz9f5r5auq14qsnsd44fZ:~# kill 4030619
```

👉 解释一下：

- `nohup`：关终端也不死
- `> cf.log`：输出写到日志
- `2>&1`：错误也写进去

👉 查看日志：

```
tail -f cf.log
```

🌐 成功后你会看到

类似：

```shell
https://xxxxx.trycloudflare.com
```

 👉 用这个地址打开你的项目
 👉 **HTTPS 自动搞定**
 👉 摄像头 / 麦克风 API 就能用了 ✅

#  影视api接口

> 类似于**https://jszyapi.com/api.php/provide/vod**

这个接口标准名称叫**苹果 CMS 影视采集 API**，也常被称作：

1. **电影天堂影视数据源接口**（域名含`dytt`电影天堂标识）
2. **maccms 通用 VOD 视频接口**（适配苹果 CMS10 标准接口格式）
3. 圈内俗称**影视 JSON 采集接口**、**资源站聚合接口**

### 补充说明

- 接口路径 `/api.php/provide/vod` 是**苹果 CMS 官方固定接口路由**，专门用来输出影视列表、分类、详情 JSON 数据。
- 域名 `caiji.dyttzyapi.com` 是第三方搭建的**电影天堂资源采集分站**，专门对外提供影视源数据。
- 常用于影视 APP、影视网站、TV 盒子导入**自定义数据源**。
