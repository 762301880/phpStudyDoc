# Redis 缓存穿透 & 缓存雪崩 解决方案（PHP 实战版）

## 一、缓存穿透（查不存在的数据）

### 什么是穿透？

用户查询**数据库里根本不存在**的数据（比如 id=-1），缓存查不到，直接请求数据库，大量请求会打崩数据库。

### 解决方案（2 种最实用）

1. **缓存空值**：查不到数据时，把 `null/空值` 存入 Redis，设置**短过期时间**（避免长期占用内存）
2. **布隆过滤器**：提前把合法 ID 存入过滤器，请求先过过滤器，不合法直接拒绝（高性能）

------

### 1. 缓存空值（PHP 最简实现）

```php
<?php
$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

function getUserInfo($userId) {
    global $redis;
    $cacheKey = "user:info:" . $userId;

    // 1. 查缓存
    $cacheData = $redis->get($cacheKey);
    if ($cacheData !== false) {
        // 缓存是空值，直接返回，不查库
        return $cacheData === 'null' ? null : json_decode($cacheData, true);
    }

    // 2. 缓存未命中，查数据库
    $user = queryDB($userId);

    // 3. 缓存空值 + 过期时间（关键！）
    if (empty($user)) {
        // 不存在的数据，缓存空字符串/null，过期 60s 防攻击
        $redis->setex($cacheKey, 60, 'null');
        return null;
    }

    // 4. 正常数据缓存
    $redis->setex($cacheKey, 3600, json_encode($user));
    return $user;
}

// 模拟数据库查询
function queryDB($userId) {
    // 假设只有 id=100 存在
    return $userId == 100 ? ['id' => 100, 'name' => 'test'] : null;
}
```

------

### 2. 布隆过滤器（防穿透终极方案）

**原理**：提前把所有合法用户 ID 存入布隆过滤器，请求先校验，不存在直接返回。

PHP 推荐用 **Redis 布隆过滤器（RedisBloom）**，无需自己实现算法。

```php
<?php
$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

// 初始化：把所有合法用户ID加入布隆过滤器（项目启动时执行一次）
function initBloomFilter() {
    global $redis;
    $validUserIds = [100, 101, 102]; // 从数据库查出所有合法ID
    foreach ($validUserIds as $id) {
        $redis->rawCommand('BF.ADD', 'user:bloom:filter', $id);
    }
}

function getUserInfoBloom($userId) {
    global $redis;
    
    // 布隆过滤器校验：不存在直接返回
    $exists = $redis->rawCommand('BF.EXISTS', 'user:bloom:filter', $userId);
    if (!$exists) {
        return null;
    }

    // 正常查缓存 + 数据库逻辑
    $cacheKey = "user:info:" . $userId;
    $cacheData = $redis->get($cacheKey);
    if ($cacheData) return json_decode($cacheData, true);

    $user = queryDB($userId);
    if ($user) $redis->setex($cacheKey, 3600, json_encode($user));
    return $user;
}
```

## 二、缓存雪崩（大量缓存同时过期）

### 什么是雪崩？

大量缓存**同一时间过期**，所有请求同时打数据库 → 数据库瞬间崩溃。

### 核心原因

- 大量 key 用**相同过期时间**
- Redis 宕机

### 解决方案（3 种企业标配）

1. **过期时间加随机值**：避免集体过期
2. **互斥锁（分布式锁）**：缓存过期时，只允许一个请求查库更新
3. **缓存永不过期 + 后台定时更新**：彻底杜绝过期

------

### 1. 过期时间 + 随机值（最简单有效）

```php
<?php
$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

function getGoodsInfo($goodsId) {
    global $redis;
    $cacheKey = "goods:info:" . $goodsId;

    $data = $redis->get($cacheKey);
    if ($data) return json_decode($data, true);

    // 查数据库
    $goods = queryGoodsDB($goodsId);
    
    // 核心：基础时间 + 随机秒数（3600±300）
    $baseExpire = 3600;
    $randomExpire = rand(1, 300);
    $expire = $baseExpire + $randomExpire;

    if ($goods) {
        $redis->setex($cacheKey, $expire, json_encode($goods));
    }
    return $goods;
}
```

------

### 2. 分布式锁（Redis 互斥锁）防雪崩

**原理**：缓存过期时，**只允许一个请求去查数据库**，其他请求等待重试，避免压垮数据库。

```php
<?php
$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

function getGoodsInfoWithLock($goodsId) {
    global $redis;
    $cacheKey = "goods:info:" . $goodsId;
    $lockKey = "lock:goods:" . $goodsId;

    // 1. 查缓存
    $data = $redis->get($cacheKey);
    if ($data) return json_decode($data, true);

    // 2. 尝试获取分布式锁（5秒超时）
    $lock = $redis->set($lockKey, 1, ['nx', 'ex' => 5]);
    if (!$lock) {
        // 未获取锁，等待 100ms 重试
        usleep(100000);
        return getGoodsInfoWithLock($goodsId);
    }

    try {
        // 3. 双重检查缓存（防止并发重复查库）
        $data = $redis->get($cacheKey);
        if ($data) return json_decode($data, true);

        // 4. 查数据库
        $goods = queryGoodsDB($goodsId);
        
        // 5. 写缓存
        $expire = 3600 + rand(1, 300);
        $redis->setex($cacheKey, $expire, json_encode($goods));
        return $goods;
    } finally {
        // 6. 释放锁
        $redis->del($lockKey);
    }
}
```

### 3. 缓存永不过期（终极方案）

- 缓存**不设置过期时间**
- 把过期时间存在 value 里
- 后台定时任务 / 请求发现快过期时，**异步更新缓存**

```php
<?php
$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

function getGoodsInfoNoExpire($goodsId) {
    global $redis;
    $cacheKey = "goods:info:" . $goodsId;

    $cache = $redis->get($cacheKey);
    if ($cache) {
        $data = json_decode($cache, true);
        // 检查是否过期
        if ($data['expire_time'] > time()) {
            return $data['data'];
        }
        // 异步更新缓存（不阻塞请求）
        asyncUpdateGoodsCache($goodsId);
        return $data['data']; // 先返回旧数据
    }

    // 首次查询
    $goods = queryGoodsDB($goodsId);
    $saveData = [
        'data' => $goods,
        'expire_time' => time() + 3600
    ];
    $redis->set($cacheKey, json_encode($saveData)); // 永不过期
    return $goods;
}

// 异步更新缓存
function asyncUpdateGoodsCache($goodsId) {
    // PHP 可用 exec/队列实现异步
    exec("php update_cache.php $goodsId > /dev/null 2>&1 &");
}
```

# 总结（面试 / 开发必背）

## 缓存穿透

1. **缓存空值**：简单通用，适合中小型项目
2. **布隆过滤器**：高性能，适合高并发

## 缓存雪崩

1. **过期时间 + 随机值**：最简单，必加
2. **分布式锁**：控制数据库访问并发
3. **永不过期 + 异步更新**：最稳定，大型项目首选

# 补充

## 什么是缓存雪崩

缓存雪崩：大量请求 涌向同一个 / 少数几个缓存全部失效的热点接口

### 1. 缓存雪崩是什么

**核心：大量 key 同时过期 / 缓存挂了 → 瞬间所有请求全部击穿打到数据库**

两种情况：

1）**大面积缓存同时过期（最常见）**

比如：

- 成千上万个商品缓存 **统一设置 1 小时过期**
- 时间一到，**瞬间全部失效**
- 海量请求同一时间 → 压到 DB → 数据库卡死

2）**Redis 整体宕机 / 断网**

所有接口缓存全挂，流量全部打库，直接拖垮系统。

### 2. 和你问的区分

- **缓存雪崩**

  缓存集体失效 → **一堆请求打同一批数据（可单接口、可多接口）**

  重点：**同一时间、大量流量压 DB**

- **缓存穿透**

  查**不存在的数据**，缓存永远不命中，一直打 DB

- **缓存击穿**

  **单个热点 key 失效**，瞬间打爆 DB（比如秒杀商品）

### 最简对比

- 穿透：查没有的，一直进库
- 击穿：一个热 key 过期，瞬间进库
- 雪崩：一堆 key 同时过期，大批量进库

### 3. 怎么防（最实用 3 招）

1. 过期时间加随机值

   避免同时过期：

   ```
   ttl = 3600 + 随机 60~300
   ```

2. Redis 高可用集群

   主从 + 哨兵 / 集群，防止缓存挂

3. 限流 + 互斥锁 / 布隆过滤器

   挡住无效流量、防并发击穿
