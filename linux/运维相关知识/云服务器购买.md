# 个人小站点配置服务器

```shell
配置信息
规格
2核 | 2GiB

云硬盘
40GiB(系统盘) 高IO

带宽
2 Mbit/s

镜像信息
Ubuntu 22.04 server 64bit
```

作为 Web 开发者（用 FastAdmin 框架），你可能需要部署项目到云服务器，这些配置是云服务器的核心参数，我用 **大白话 + 开发场景关联** 逐一解释，确保你能直接理解并判断是否适合你的需求：

### 一、规格：2 核 | 2GiB（服务器的 “大脑 + 内存”）

#### 1. 通俗解释：

- **2 核**：指服务器的 CPU（中央处理器）核心数，相当于 “工人数量”——2 个工人同时干活。
- **2GiB**：指服务器的内存（RAM），相当于 “工人的工作台大小”—— 工作台越大，能同时摆放的工具（运行的程序、数据）越多。

#### 2. 开发场景关联（FastAdmin 项目适用）：

- 适合：个人开发、小型项目（比如管理系统、接口服务）、测试环境。
  - 能同时运行：FastAdmin（PHP+MySQL）+ Nginx/Apache 服务器，支持 **10-50 人同时访问**（如果是后台管理系统，完全够用）。
- 不适合：高并发场景（比如电商网站、大量用户同时提交数据），会出现 “卡慢”（CPU 不够用）或 “内存溢出”（程序崩溃）。

### 二、云硬盘：40GiB (系统盘) 高 IO（服务器的 “硬盘”）

#### 1. 通俗解释：

- **40GiB**：存储容量（1GiB≈1G），相当于 “文件柜大小”—— 用来装系统、软件、你的 FastAdmin 项目代码、数据库数据。
- **系统盘**：这是服务器的 “主硬盘”，必须有，用来安装操作系统（比如后面的 Ubuntu），不能删除。
- **高 IO**：硬盘的读写速度（IO = 输入输出），“高 IO”≈“高速硬盘”，比普通硬盘快，适合运行数据库（比如 MySQL）、频繁读写文件的场景。

#### 2. 开发场景关联：

- 40GiB 足够用：Ubuntu 系统占 10-15GiB，FastAdmin 项目 + MySQL 数据库占 5-10GiB，剩余空间足够存放日志、备份文件。
- 高 IO 很重要：FastAdmin 依赖 MySQL 数据库，查询数据时需要硬盘快速读写，“高 IO” 能让你的管理系统页面加载更快、数据查询更流畅。
- 扩展：如果以后项目变大（比如数据量超 20GiB），可以额外加 “数据盘”（相当于额外的文件柜），不用换系统盘。

### 三、带宽：2 Mbit/s（服务器的 “网络速度”）

#### 1. 通俗解释：

- 带宽是服务器和互联网之间的 “数据传输通道宽度”，相当于 “水管粗细”——2 Mbit/s 是小水管，100 Mbit/s 是大水管。
- 注意单位：**Mbit/s（兆比特 / 秒）≠ MB/s（兆字节 / 秒）**，换算公式：1 MB/s = 8 Mbit/s，所以 2 Mbit/s ≈ 0.25 MB/s（每秒传输约 256KB 数据）。

#### 2. 开发场景关联：

- 适合：开发 / 测试环境、小型后台管理系统（访问人数少、不传输大文件）。
  - 比如：FastAdmin 后台加载页面（数据量小，约几十 KB），加载时间≈0.2 秒，完全流畅；
  - 传输 10MB 的项目文件到服务器，需要时间≈10MB / 0.25MB/s = 40 秒（能接受）。
- 不适合：对外提供下载服务、传输大文件（比如视频、压缩包）、大量用户同时访问（会出现 “加载慢”“超时”）。
- 扩展：如果以后项目上线后访问量变大，可以随时升级带宽（比如升到 10 Mbit/s，相当于 0.125 MB/s，传输速度翻倍）。

### 四、镜像信息：Ubuntu 22.04 server 64bit（服务器的 “操作系统”）

#### 1. 通俗解释：

- 镜像 = 操作系统的 “安装包”，云服务器启动时会自动安装这个系统，相当于给服务器装 “Windows” 或 “MacOS”。
- **Ubuntu**：一个流行的 Linux 系统（开源、免费、稳定，适合服务器）；
- **22.04**：系统版本号（LTS 长期支持版，支持 5 年，不用频繁升级）；
- **server**：服务器版（没有图形界面，只有命令行，占用资源少，适合远程操作）；
- **64bit**：64 位系统（支持大内存，能运行 PHP 7+、MySQL 8 + 等新版本软件，和 FastAdmin 完全兼容）。
