# 分销系统核心字段设计（通用版）

我给你整理**企业级、可直接落地、兼容电商 / 零售 / SAAS**的分销系统字段设计，包含**用户、分销关系、订单、佣金、提现、商品**6 大核心模块，无冗余、可直接建表使用。

## 一、整体设计原则

1. **层级清晰**：支持 1~3 级分销（合规主流）
2. **关系可追溯**：上级、团队、来源链路完整
3. **佣金可结算**：订单、佣金、提现闭环
4. **防篡改**：关键金额、状态不可随意修改
5. **易扩展**：支持等级、权限、活动

# 二、核心表结构 + 字段设计（MySQL 风格）

## 1. 用户表（user）

存储所有普通用户 + 分销商

```sql
CREATE TABLE `user` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `nickname` varchar(50) DEFAULT '' COMMENT '昵称',
  `mobile` varchar(20) DEFAULT '' COMMENT '手机号',
  `avatar` varchar(255) DEFAULT '' COMMENT '头像',
  `is_distributor` tinyint DEFAULT 0 COMMENT '是否分销商 0否 1是',
  `distributor_level` int DEFAULT 0 COMMENT '分销商等级 0普通 1一级 2高级',
  `balance` decimal(12,2) DEFAULT 0.00 COMMENT '可提现余额',
  `frozen_amount` decimal(12,2) DEFAULT 0.00 COMMENT '冻结金额',
  `total_commission` decimal(12,2) DEFAULT 0.00 COMMENT '累计佣金',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  `updated_at` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_mobile` (`mobile`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 2. 分销关系表（distributor_relation）

**分销系统最核心表**：记录谁推荐谁、上下级关系、团队链路

```sql
CREATE TABLE `distributor_relation` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `user_id` bigint NOT NULL COMMENT '当前用户ID',
  `parent_id` bigint NOT NULL COMMENT '直接上级ID',
  `level_1_id` bigint DEFAULT 0 COMMENT '1级上级',
  `level_2_id` bigint DEFAULT 0 COMMENT '2级上级',
  `level_3_id` bigint DEFAULT 0 COMMENT '3级上级',
  `team_path` varchar(255) DEFAULT '' COMMENT '团队链路 0-1001-1002-1003',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_user_id` (`user_id`),
  KEY `idx_parent_id` (`parent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='分销上下级关系表';
```

## 3. 分销订单表（distributor_order）

订单产生后，自动生成分销佣金记录

```sql
CREATE TABLE `distributor_order` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `order_no` varchar(64) NOT NULL COMMENT '订单号',
  `user_id` bigint NOT NULL COMMENT '下单用户ID',
  `order_amount` decimal(12,2) NOT NULL COMMENT '订单实付金额',
  `commission_rate` decimal(6,2) DEFAULT 0.00 COMMENT '佣金比例',
  `total_commission` decimal(12,2) DEFAULT 0.00 COMMENT '订单总佣金',
  `status` tinyint DEFAULT 0 COMMENT '0待结算 1已结算 2已取消',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  `settle_time` datetime DEFAULT NULL COMMENT '结算时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_order_no` (`order_no`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT=utf8mb4 COMMENT='分销订单表';
```

## 4. 佣金明细表（commission_record）

**每一笔佣金给谁、多少、什么状态**

```sql
CREATE TABLE `commission_record` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `order_no` varchar(64) NOT NULL COMMENT '订单号',
  `user_id` bigint NOT NULL COMMENT '获得佣金的用户',
  `parent_level` int DEFAULT 0 COMMENT '1级 2级 3级',
  `commission_amount` decimal(12,2) DEFAULT 0.00 COMMENT '佣金金额',
  `status` tinyint DEFAULT 0 COMMENT '0待结算 1已结算 2已失效',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  `settle_time` datetime DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_order_no` (`order_no`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='佣金明细表';
```

## 5. 提现申请表（withdraw_apply）

分销商提现流水

```sql
CREATE TABLE `withdraw_apply` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `withdraw_no` varchar(64) NOT NULL COMMENT '提现单号',
  `user_id` bigint NOT NULL COMMENT '申请人ID',
  `amount` decimal(12,2) NOT NULL COMMENT '提现金额',
  `service_fee` decimal(12,2) DEFAULT 0.00 COMMENT '手续费',
  `real_amount` decimal(12,2) NOT NULL COMMENT '实际到账',
  `status` tinyint DEFAULT 0 COMMENT '0待审核 1已打款 2驳回',
  `pay_type` varchar(32) DEFAULT '' COMMENT '支付宝/微信/银行卡',
  `pay_account` varchar(100) DEFAULT '' COMMENT '收款账号',
  `apply_time` datetime DEFAULT CURRENT_TIMESTAMP,
  `audit_time` datetime DEFAULT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_withdraw_no` (`withdraw_no`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='提现申请表';
```

## 6. 商品分销配置表（product_distribution）

给商品单独设置分销佣金

```sql
CREATE TABLE `product_distribution` (
  `id` bigint NOT NULL AUTO_INCREMENT,
  `product_id` bigint NOT NULL COMMENT '商品ID',
  `level_1_rate` decimal(6,2) DEFAULT 0.00 COMMENT '1级佣金%',
  `level_2_rate` decimal(6,2) DEFAULT 0.00 COMMENT '2级佣金%',
  `level_3_rate` decimal(6,2) DEFAULT 0.00 COMMENT '3级佣金%',
  `is_enabled` tinyint DEFAULT 1 COMMENT '是否开启分销',
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_product_id` (`product_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='商品分销佣金配置';
```

# 三、核心业务逻辑（极简说明）

1. **用户注册 → 写入分销关系表**（绑定上级）
2. **用户下单 → 生成分销订单 → 计算佣金**
3. **佣金按 1/2/3 级分别入账**
4. **订单完成 → 佣金解冻 → 可提现**
5. **分销商申请提现 → 冻结余额 → 审核打款**
