## 抽象工厂设计模式

PHP「抽象工厂模式（Abstract Factory Pattern）」示例。
 场景：不同品牌生产不同类型的产品，比如：

- 苹果工厂：生产 苹果手机、苹果电脑
- 华为工厂：生产 华为手机、华为电脑

保证“同一产品族”的一致性。

### 抽象工厂模式结构

主要角色：

| 角色            | 作用         |
| --------------- | ------------ |
| AbstractFactory | 抽象工厂接口 |
| ConcreteFactory | 具体工厂     |
| AbstractProduct | 抽象产品     |
| ConcreteProduct | 具体产品     |
| Client          | 客户端       |

### PHP 示例

```php
<?php

/**
 * =========================
 * 1. 抽象产品：手机接口
 * =========================
 */
interface Phone
{
    public function call();
}

/**
 * =========================
 * 2. 抽象产品：电脑接口
 * =========================
 */
interface Computer
{
    public function work();
}

/**
 * =========================
 * 3. 具体产品：苹果手机
 * =========================
 */
class IPhone implements Phone
{
    public function call()
    {
        echo "使用 iPhone 打电话\n";
    }
}

/**
 * =========================
 * 4. 具体产品：华为手机
 * =========================
 */
class HuaweiPhone implements Phone
{
    public function call()
    {
        echo "使用 华为手机 打电话\n";
    }
}

/**
 * =========================
 * 5. 具体产品：Mac电脑
 * =========================
 */
class MacComputer implements Computer
{
    public function work()
    {
        echo "使用 Mac 电脑办公\n";
    }
}

/**
 * =========================
 * 6. 具体产品：华为电脑
 * =========================
 */
class HuaweiComputer implements Computer
{
    public function work()
    {
        echo "使用 华为电脑办公\n";
    }
}

/**
 * =========================
 * 7. 抽象工厂
 *
 * 规定：
 * 每个工厂都必须能够：
 * - 生产手机
 * - 生产电脑
 * =========================
 */
interface AbstractFactory
{
    /**
     * 创建手机
     */
    public function createPhone(): Phone;

    /**
     * 创建电脑
     */
    public function createComputer(): Computer;
}

/**
 * =========================
 * 8. 苹果工厂
 * =========================
 */
class AppleFactory implements AbstractFactory
{
    /**
     * 创建苹果手机
     */
    public function createPhone(): Phone
    {
        return new IPhone();
    }

    /**
     * 创建苹果电脑
     */
    public function createComputer(): Computer
    {
        return new MacComputer();
    }
}

/**
 * =========================
 * 9. 华为工厂
 * =========================
 */
class HuaweiFactory implements AbstractFactory
{
    /**
     * 创建华为手机
     */
    public function createPhone(): Phone
    {
        return new HuaweiPhone();
    }

    /**
     * 创建华为电脑
     */
    public function createComputer(): Computer
    {
        return new HuaweiComputer();
    }
}

/**
 * =========================
 * 10. 客户端代码
 * =========================
 */

// 创建苹果工厂
$appleFactory = new AppleFactory();

// 生产苹果产品
$iphone = $appleFactory->createPhone();
$mac = $appleFactory->createComputer();

// 使用产品
$iphone->call();
$mac->work();

echo "-----------------\n";

// 创建华为工厂
$huaweiFactory = new HuaweiFactory();

// 生产华为产品
$huaweiPhone = $huaweiFactory->createPhone();
$huaweiComputer = $huaweiFactory->createComputer();

// 使用产品
$huaweiPhone->call();
$huaweiComputer->work();
```

### 输出结果

```php
使用 iPhone 打电话
使用 Mac 电脑办公
-----------------
使用 华为手机 打电话
使用 华为电脑办公
```

### 抽象工厂模式核心思想

抽象工厂最大的特点：

> 一整个“产品族”一起创建

比如：

| 工厂          | 手机        | 电脑           |
| ------------- | ----------- | -------------- |
| AppleFactory  | iPhone      | Mac            |
| HuaweiFactory | HuaweiPhone | HuaweiComputer |

客户端根本不需要关心具体类名：

```
$factory->createPhone();
$factory->createComputer();
```

直接切换整个产品族。

和工厂方法模式区别

很多人容易混。

工厂方法模式

只生产一种产品：

```
PhoneFactory
```

只管手机。

抽象工厂模式

生产一组关联产品：

```php
AppleFactory
```

既生产：

- Phone
- Computer

属于“产品族”。

### 适用场景

适合：

#### 1. 多产品族

例如：

- 不同数据库驱动
- 不同 UI 风格
- 不同品牌设备

####  2. 需要保持产品兼容性

例如：

```php
苹果鼠标 + 苹果键盘 + 苹果电脑
```

一套生态。

### 实际开发中的经典应用

例如 Laravel / Symfony 里的：

- 数据库驱动工厂
- Cache Driver
- Queue Driver
- 不同云厂商 SDK

本质都带有抽象工厂思想。

### 优点

- 解耦
- 易扩展
- 产品族统一
- 符合开闭原则

### 缺点

新增“产品类型”会比较麻烦。

比如新增：

```php
Tablet
```

所有工厂都得改。

### 一句话记忆

可以这样记：

```php
工厂方法：一个工厂生产一种产品
抽象工厂：一个工厂生产一整套产品
```
