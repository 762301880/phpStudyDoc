#  php协程与线程的区别

## 一、先一句话总结

- **线程**：操作系统管理的**真正并行**，重量级、开销大、有锁竞争、PHP 原生支持弱（需要扩展）。
- **协程**：用户态代码管理的**轻量级切换**，**单线程内并发**，无锁、开销极小、是 Swoole/Workerman 等主流方案。

## 二、核心区别对照表

| 对比项        | 线程（Thread）                        | 协程（Coroutine）            |
| ------------- | ------------------------------------- | ---------------------------- |
| **管理者**    | 操作系统内核                          | 用户代码（PHP 框架 / 扩展）  |
| **切换开销**  | 非常大（保存 CPU 寄存器、内核态切换） | 极小（仅保存栈 / 上下文）    |
| **并行能力**  | 真正多核并行                          | 单线程内并发（非并行）       |
| **资源占用**  | 高（一个线程≈MB 级内存）              | 极低（一个协程≈KB 级内存）   |
| **锁 / 竞争** | 需要加锁（mutex），容易死锁           | 单线程执行，**完全不需要锁** |
| **PHP 支持**  | 原生不支持，需安装`pthreads`扩展      | Swoole/Workerman 原生支持    |
| **稳定性**    | 复杂，易出问题                        | 简单、稳定、可控             |
| **数量上限**  | 几百个就卡顿                          | 轻松创建**上万 +**           |

## 三、通俗比喻

### 1. 线程 = 多个工人同时干活

- 有**多个 CPU 核心**，真的同时干多件事。
- 但工人之间**抢工具、抢桌子**，必须加锁管制。
- 招聘 / 解雇工人成本很高（创建 / 销毁线程开销大）。

### 2. 协程 = 一个工人快速切换任务

- 只有**一个工人**，但手脚极快。
- 烧水时不等水开，**立刻去切菜**；水开了再回来。
- 工人自己控制切换，**不用跟老板（操作系统）申请**。
- 不用抢工具，**完全不冲突**。

## 四、PHP 环境下的关键差异（重点！）

### 1. 线程在 PHP 中几乎不用

- PHP 官方**不推荐、不原生支持**多线程。
- 唯一扩展 `pthreads` 只支持 **CLI 模式**，不支持 FPM。
- 共享变量必须加锁，极易死锁、崩溃。
- 社区生态几乎废弃。

### 2. 协程是 PHP 高并发的标准方案

- **Swoole、Workerman、Hyperf** 全是协程。
- 单进程可跑 **1W+ 协程**，支撑高并发网络请求。
- 同步写法，异步性能，代码简单无锁。
- 支持：MySQL、Redis、Curl、文件 IO 自动协程化。

#  协程只能在cli中使用

主流 PHP 协程（Swoole/OpenSwoole）只能在 CLI 模式下运行，不能在传统 FPM 网页模式下使用。

### 为什么 FPM 不能用协程

1. **运行机制冲突**

   

   - FPM：

     短生命周期、多进程、同步阻塞

     - 一次请求 → 一个进程 → 处理完就销毁
     - 没有**事件循环 (EventLoop)** 和**协程调度器**

   - 协程：需要常驻内存、单进程、非阻塞 IO、主动调度

     - 必须在一个**长期运行的进程**里才能切换、挂起、恢复

2. **直接报错**

   在 FPM 里写 `go()` / `Co::sleep()` 会直接报：

   ```php
   Fatal error: Uncaught Swoole\Error: No coroutines running
   ```

哪些场景可以用协程

1. **Swoole HTTP 服务器**（替代 FPM）

```php
$server = new Swoole\Http\Server('0.0.0.0', 9501);
$server->on('Request', function ($req, $resp) {
    go(function () {
        // 协程内处理请求
        $resp->end('Hello Swoole');
    });
});
$server->start();
```

用 `php server.php` 启动，**完全脱离 FPM**。

2. CLI 脚本 / 定时任务

```php
Co\run(function () {
    go(function () { /* 任务1 */ });
    go(function () { /* 任务2 */ });
});
```
