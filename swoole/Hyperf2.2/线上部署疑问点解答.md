## 线上部署能不能用热更新[watcher](https://www.hyperf.wiki/2.2/#/zh-cn/watcher)

###  核心（重）

1. **本地开发环境**

   ✅ **强烈推荐使用 Watcher 热更新**

   修改代码自动重启，不用手动重启服务，开发效率极高

   

2. **线上生产环境**

   ❌ **绝对禁止使用 Watcher 热更新**

   生产必须**关闭热更新**，否则会导致：

   - 性能损耗
   - 代码异常、服务不稳定
   - 内存泄漏
   - 线上安全风险

### 开发环境怎么开启热更新（最简单）

```php
composer require hyperf/watcher --dev
```

启动命令改用

```php
php bin/hyperf.php server:watch
```

只要修改代码，**自动重启服务**，路由、控制器修改立刻生效。

### 线上生产环境 正确部署方式

1. 启动命令（生产标准）

```bash
php bin/hyperf.php start -d
```

`-d` = 守护进程后台运行

2. 生产更新代码流程

1. 上传新代码

2. 平滑重启

   （不中断请求）

   ```bash
   php bin/hyperf.php reload
   ```

   或者

   ```bash
   kill -USR1 主进程ID
   ```

✅ **生产唯一正确方案：代码更新 + 平滑重启**

不要用热更新！

### 为什么生产不能用热更新？

Hyperf 是**常驻内存框架**

热更新是**开发工具**，不是生产工具

文件监听会消耗服务器资源

生产环境代码不应该频繁变动，变动必须走平滑重启
